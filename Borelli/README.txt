# Repositorio Colaborativo de Trabajos Prácticos de Programación

¡Bienvenidos al repositorio colaborativo de la materia de programación! Este repositorio está diseñado para ayudar a los estudiantes a trabajar juntos en los trabajos prácticos de la materia, compartiendo conocimientos y recursos.

## ¿Cómo Contribuir?

1. **Forkea el Repositorio:** Haz clic en el botón "Fork" en la esquina superior derecha de esta página para crear tu propia copia del repositorio en tu cuenta de GitHub.

2. **Clona el Repositorio:** Clona tu fork del repositorio a tu computadora local utilizando el comando:


3. **Crea una Rama:** Antes de hacer cualquier cambio, crea una nueva rama para trabajar en ella. Esto ayuda a mantener ordenado el flujo de trabajo. Utiliza un nombre descriptivo para la rama, como "agregar-trabajo-X" o "corregir-trabajo-Y".


4. **Realiza Cambios:** Agrega tus archivos y realiza las modificaciones necesarias.

5. **Commitea y Empuja:** Después de hacer tus cambios, commitea tus cambios y súbelos a tu repositorio en GitHub.


6. **Solicita una Pull Request:** En tu repositorio en GitHub, crea una Pull Request (PR) para que tus cambios sean incorporados al repositorio principal. Describe tus cambios y menciona el número de trabajo práctico al que se refiere.

## Estructura del Repositorio

El repositorio está organizado de la siguiente manera:

- `/trabajos`: En esta carpeta encontrarás subcarpetas numeradas correspondientes a cada trabajo práctico.
- `/trabajo-1`
- `/trabajo-2`
- ...

Dentro de cada carpeta de trabajo práctico, puedes incluir los archivos, código fuente, recursos y cualquier otra cosa necesaria para completar el trabajo.

## Normas de Colaboración

- Mantén un lenguaje respetuoso y profesional en todas las interacciones.
- Siempre trabaja en ramas separadas para cada tarea y evita hacer cambios directamente en la rama principal (master/main).
- Coordina con otros colaboradores para evitar duplicación de esfuerzos.
- Las decisiones importantes sobre el repositorio serán tomadas a través de discusiones en las Pull Requests.
- No dudes en utilizar la sección de problemas (Issues) para discutir problemas específicos o hacer preguntas.

¡Esperamos que este repositorio facilite la colaboración y el aprendizaje en la materia de programación! Si tienes preguntas, no dudes en abrir un issue o contactar a los administradores del repositorio.
